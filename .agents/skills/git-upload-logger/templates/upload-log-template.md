# 上传批次日志模板

默认文件：

- `runtime/logs/YYYY-MM-DD.md`

写入要求：

- 同一天的上传批次追加到同一个文件
- 一次上传批次写一个独立分段
- `commit hash`、推送区间和最终上传结果只写真实 Git 结果

建议骨架：

```md
## 上传批次 YYYY-MM-DD HH:MM

### 当前任务 / 阶段
- 当前任务：
- 当前阶段：

### 上传批次范围
- 本次上传边界：

### 关键操作
- 本次执行了什么：

### 改动范围
- 涉及的文件 / 目录 / 模块：

### 验证情况
- 已做验证：
- 未验证项：

### 上传目标与实际结果
- 目标远端 / 分支：
- 实际结果：
- `commit hash` / 推送区间（如已知且来自 Git）：

### 剩余风险
- 当前风险：

### 未完成项
- 下一步：
```

使用要求：

- 不得擅自缩小用户已明确的上传边界
- 没有承接快照时，退化为读取当前会话与真实 Git 结果
- 失败、部分成功和阻塞情况都要照实记录
