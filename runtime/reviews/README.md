# reviews

本目录用于正式 review 记录。  
它是运行对象，不属于 `docs/` 说明层。  

## 什么时候写入

- 中改及以上任务
- 高风险任务
- 发布前正式审查
- AI 行为变更且需要正式 review 留痕

## 文件建议

- 命名：`YYYY-MM-DD-任务名-review.md`
- 一次正式 review 对应一个记录文件
- 当前审查基准继续以 `../code_review.md` 为准，归档记录再进入本目录

## 最小内容

- 日期
- 任务名称
- 审查范围
- `Findings`
- `Risks not fully verified`
- `Evidence checked`
- `Final recommendation`

## 使用边界

- 不命中正式审查条件时，不为了“看起来完整”而创建空文件
- review 记录写事实、证据和结论，不复制聊天纪要
