# errors

本目录用于错误、返工和判断失误记录。  
它是运行对象，不属于 `docs/` 说明层。  

## 什么时候写入

- 出现明确判断失误
- 出现返工原因需要留痕
- 同类错误反复出现，值得沉淀成规则升级输入

## 文件建议

- 命名：`YYYY-MM-DD-问题名.md`
- 一次可复用错误模式对应一个文件

## 最小内容

- 问题现象
- 触发场景
- 根因判断
- 影响范围
- 修复动作
- 防再发建议

## 使用边界

- 一次性小偏差且无复用价值的问题，不必机械建档
- 当同类问题持续出现时，再把结论升级回 workflow、skill、shared protocol 或脚本
