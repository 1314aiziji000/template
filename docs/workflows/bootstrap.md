# bootstrap

## 定位

本文件只描述初始化阶段骨架。
具体执行步骤、删除顺序和输入校验由 `project-bootstrap` 负责。
本文件只回答进入条件、阶段顺序、必备产物和下一跳，不写 `project-bootstrap` 内部规则。

## 进入条件

- 当前仓是刚复制出的 `V3` 项目副本
- 根级仍存在 `项目说明书.md`
- `.agents/skills/project-bootstrap/` 仍存在
- 用户输入 `开始`，或明确要求初始化当前项目

## 本阶段回答什么

- 当前仓是否真的是首次接入模板
- 当前应采用哪一种初始化模式
- 哪些 Optional Packs 保留，哪些应裁剪
- 根级入口与目录要如何适配

## 阶段骨架

1. 读取一次性输入与根级入口
2. 判断初始化模式和 Optional Packs 启用范围
3. 给出初始化建议，并等待用户确认
4. 按确认结果适配根级入口与目录
5. 将初始化结论写入 `runtime/PLANS.md`
6. 删除未启用的目录与一次性资产
7. 将项目送回标准主链

## 本阶段必须产出

- 当前初始化模式结论
- 已启用 Packs 清单
- 根级入口适配结论
- `runtime/PLANS.md` 初始化结论
- 一次性资产删除结果

没有这 5 项，就不算初始化完成。

## 退出去向

- 初始化完成：回到日常主链
- 信息不足或确认未完成：继续停留在 bootstrap 阶段
