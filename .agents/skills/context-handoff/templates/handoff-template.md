# 承接快照模板

固定开头：

`当前承接快照如下，下一窗口可直接继续：`

建议骨架：

```md
当前承接快照如下，下一窗口可直接继续：

**当前任务**
- 当前正在处理什么：

**前面窗口阶段累计**
1. 已完成的阶段结论：
2. 已沉淀的关键决定：
3. 已确认的边界或约束：

**当前窗口阶段**
- 本窗口关键操作：
- 本窗口改动范围：
- 本窗口新增判断：

**验证情况**
- 已做验证：
- 已通过项：
- 未验证项：

**当前理解**
- 剩余风险：
- 未完成项：
- 下一窗口最可能继续的事：

**上传节点（如适用）**
- 上传目标：
- 上传结果：
- 当前已知 Git 状态边界：
```

使用要求：

- 内容按累计方式整理，不只总结当前窗口
- 质量不能低于后续补日志所需信息标准
- 如果当前任务涉及上传节点，必须补齐“上传节点”部分
